# Data Sources Pipeline — Design

## Overview

Centralized, incremental data pipeline for ChatVote. All data sources live in the backend as services, configurable and observable from an admin UI. Each node tracks what it processed and only acts on new/changed data.

## Architecture

```
Frontend: /admin/data-sources/[secret]
    │  REST calls with X-Admin-Secret header
    ▼
Backend: /api/v1/admin/data-sources/*
    │  Reads/writes pipeline config
    ▼
Firestore: data_pipeline_config/{node_id}
    │  enabled, status, last_run, counts, settings, checkpoints
    ▼
src/services/data_pipeline/
    ├── base.py           — DataSourceNode ABC + registry
    ├── candidatures.py   — data.gouv.fr CSV (candidates + lists)
    ├── population.py     — INSEE population CSV
    ├── websites.py       — Google Sheet sync via Drive API + SA
    ├── professions.py    — PDF scraper from interieur.gouv.fr
    ├── seed.py           — Merge all sources → Firestore collections
    └── indexer.py        — Chunk + embed → Qdrant (costly, disabled by default)
```

## Pipeline DAG

```
[Candidatures]   [Population]   [Websites]
  data.gouv.fr     INSEE CSV     Google Sheet
       └──────────────┼──────────────┘
                      ▼
              [Seed / Merge]  ←── [Professions de foi]
              Firestore sync       PDF scraper
                      │
                      ▼
              [Qdrant Indexer]
              chunk + embed
```

## Node model (Firestore: `data_pipeline_config/{node_id}`)

```json
{
  "node_id": "candidatures",
  "label": "Candidatures CSV",
  "enabled": true,
  "status": "idle | running | success | error",
  "last_run_at": "2026-03-10T16:00:00Z",
  "last_duration_s": 12.3,
  "last_error": null,
  "counts": { "total_rows": 809213, "matched_communes": 286, "candidates": 1603 },
  "settings": { "top_communes": 287 },
  "checkpoints": {
    "source_hash": "sha256:abc123",
    "source_modified": "2026-03-06T16:00:00Z",
    "processed_keys": []
  }
}
```

## Incremental logic per node

| Node | Change detection | Skip condition |
|------|-----------------|----------------|
| Candidatures | HTTP `Last-Modified` header or content SHA-256 | Same hash → skip download + parse |
| Population | Content SHA-256 | Same hash → skip |
| Websites | Drive API `modifiedTime` metadata (free, no download) | Same timestamp → skip |
| Professions de foi | Per-commune `last_scraped_at` in checkpoints | Commune already scraped → skip |
| Seed (merge) | Per-doc content hash before Firestore write | Unchanged doc → skip `set()` |
| Qdrant indexer | Per-candidate source content hash | Same content → skip re-embedding |

"Run now" = incremental. "Force re-run" = ignore checkpoints.

## Env vars

| Var | Purpose | Example |
|-----|---------|---------|
| `ADMIN_UPLOAD_SECRET` | Shared admin secret (existing) | `my-secret-token` |
| `GOOGLE_SHEETS_CREDENTIALS_JSON` | SA JSON string for Drive API | `{"type":"service_account",...}` |
| `GOOGLE_SHEET_ID` | Candidate websites sheet | `1JZJSQSOwlMWK5ZdXuEHetVxpesuuK_LI` |
| `DATA_GOUV_CANDIDATURES_URL` | CSV download URL (optional override) | `https://static.data.gouv.fr/...` |

## API endpoints

All require `X-Admin-Secret` header matching `ADMIN_UPLOAD_SECRET`.

```
GET  /api/v1/admin/data-sources/status          — all nodes status + config
POST /api/v1/admin/data-sources/run/{node_id}    — trigger node (body: {force: bool})
PUT  /api/v1/admin/data-sources/config/{node_id} — update settings (top_communes, enabled, etc.)
POST /api/v1/admin/data-sources/bust-cache        — reset all checkpoints
```

## Frontend page: `/admin/data-sources/[secret]`

- Same auth pattern as upload page (`/admin/upload/[secret]`)
- DAG visualization with 6 node cards
- Each card: status badge, last run, duration, counts, toggle, "Run" button, expandable settings
- Global controls: "Run all enabled", "Force re-run all", "Bust cache"
- Auto-refresh status every 5s while any node is running
